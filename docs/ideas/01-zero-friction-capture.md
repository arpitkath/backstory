# Backstory Zero-Friction Capture

## Priority

P0

## Idea

Backstory should capture AI-assisted development context passively instead of depending on the developer remembering to run a dump command.

## Rationale

Memory tools fail when capture adds friction. Claude Code, Cursor, Codex, and similar tools often write local transcript or log files that adapters can ingest. Manual dump should remain available, but it should be a fallback rather than the default workflow.

## Expected Outcome

Developers get useful memory without needing to think about Backstory during normal work.
